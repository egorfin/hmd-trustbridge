# Architecture — HMD TrustBridge

## Components

```
Browser (mobile)
    │
    ▼
Next.js Frontend (port 3000)
    │  POST /api/assess
    ▼
FastAPI Backend (port 8000)
    ├── scoring.py      → deterministic readiness score
    ├── llm.py          → OpenAI report generation
    └── supabase_client → session logging
    │
    ├── OpenAI API (gpt-4.1-mini by default)
    └── Supabase (session logs, prompt config)
```

## Data Flow

1. Frontend collects assessment answers
2. POST to `/api/assess` with `AssessmentInput`
3. Backend computes score deterministically (`scoring.py`)
4. Backend calls OpenAI with score + risk profile (`llm.py`)
5. Backend logs session to Supabase (anonymous)
6. Backend returns `AssessmentReport`
7. Frontend renders report components

## Key Design Decisions

- Score is deterministic — not generated by LLM
- LLM only generates narrative and tips
- No auth (MVP)
- CORS open (MVP — tighten before production)
- Docker Compose for local + demo environments
