# Pitch Notes

## Core Narrative

> TrustBridge converts parental anxiety into informed confidence.

Traditional sales: Product → Features → Purchase  
TrustBridge: Concern → Assessment → Trust → Strategy → Earned Recommendation

## Key Differentiators

1. **Not a sales tool** — the recommendation is earned, not assumed
2. **Explainable AI** — score is deterministic, LLM explains, not decides
3. **No grading the child** — the system uses deterministic scoring internally, but parents see a readiness path, not a judgement score
3. **Privacy-first** — no child profiles stored
4. **60-90 seconds** — fits a retail moment

## HMD Fuse Fit

The HMD Fuse is positioned as a phone designed for gradual digital independence — not a locked-down device, not an unrestricted smartphone. TrustBridge surfaces this fit naturally from the parent's own answers.

## Hackathon Judge Checklist

- [ ] Clone → env → docker compose → works in < 5 min
- [ ] Assessment completes in < 90 seconds
- [ ] Report is personalized and feels earned
- [ ] Score is explainable (not a black box)
- [ ] Supabase logs visible in dashboard
- [ ] No fear-based language anywhere in the UX
