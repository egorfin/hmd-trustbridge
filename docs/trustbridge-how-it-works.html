HMD TrustBridge
How It Works

An AI-powered Digital Safety Advisor that turns a QR scan into a personalised safer-smartphone plan for parents — in under 60 seconds, with no account required.

Fully client-side on Vercel Deterministic scoring · No hallucination Concern-first device recommendation
1

Landing page

The parent lands on the product page after scanning a QR code (billboard, retail shelf, or print) and taps "Check my child's phone readiness". The entire experience runs in the browser — no account, no sign-up.

2

Adaptive questionnaire

5 core questions with up to 3 dynamic follow-ups — the step list is recomputed after each answer. No page reloads, no loading spinners between steps.

QuestionWhat it captures
Child's ageMaturity baseline for scoring
First smartphone or upgrade?Experience level, onboarding need
Main intended useRisk surface (games, social, school, family…)
Biggest concernPrimary safety focus — drives device recommendation
Parent confidenceOnboarding support level needed

Follow-ups appear conditionally — e.g. a social media question only shows if the parent selected social media as a main use.

3

Deterministic scoring

On submit, a rule-based algorithm computes a readiness score (0–100). No AI is involved in this step. The score is 100% reproducible from the same inputs.

Base score: 70

+ Age delta          −30 to +10    (younger = larger penalty)
+ First phone       −5 or +5
+ App exposure     −10 to +8    (inferred from intended use)
+ Main concern     −4 to −8    (per concern selected)
+ Independence     −8 to +8    (derived from age)
+ Combo penalties −5 to −10   (e.g. young child + social media)

→ Final score clamped to 0–100

< 40not_readySafer start recommended
4

HMD device recommendation

A separate concern-first logic selects the HMD product path independently of the score. The parent's stated worry drives the recommendation — not just a number.

ConcernRecommended path

strangersHMD Fuse (age < 13) · HMD Fusion X1 (age ≥ 13)
screen_timeHMD Fusion X1
cyberbullyingHMD Fusion X1
social_pressureHMD Fusion X1
privacy / not sureDerived from readiness level

Two children with the same readiness score can receive different device paths depending on what the parent is most concerned about.

Full backend version

OpenAI GPT-4.1-mini writes these 8 fields using the full scoring context as a structured JSON prompt. Score and risk profile are never touched by the LLM.

Key design decisions

Score is never set by AI Prevents hallucination of safety-critical numbers. The LLM only writes narrative text, never touches the score.
Concern beats score for device path The parent's stated worry drives the HMD recommendation — not just a readiness number. More relevant, less generic.
No personal data collected No name, no email, no account. The assessment runs entirely in the browser session.
Works from any QR scan Vercel deployment has zero runtime dependencies. Loads fast on mobile, works on any network.
